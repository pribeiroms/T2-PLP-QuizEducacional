# Proposta do Trabalho (Valor: 3,0 pontos)

Os grupos deverão desenvolver um Sistema (a escolha do grupo) aplicando conceitos
de diferentes paradigmas de programação. O objetivo é comparar formas distintas de
resolver problemas computacionais e refletir sobre as vantagens e limitações de cada
paradigma.

# Temas Permitidos (Exemplos...cada grupo escolhe ou define
algum)

• Sistema de Biblioteca
• Sistema de Clínica Médica
• Sistema de Loja Virtual
• Sistema de Academia
• Quiz Educacional
• Jogo da Forca
• RPG Textual
• etc. etc. etc.

## Requisitos Obrigatórios

- [ ]  Grupos de até 6 (seis) integrantes.
- [ ]  Implementar pelo menos três funcionalidades principais.
- [ ]  Utilizar Programação Orientada a Objetos.
- [ ]  Utilizar pelo menos um recurso de Programação Funcional.
- [ ]  Elaborar relatório comparando os paradigmas utilizados.
- [ ]  Realizar apresentação e demonstração prática.

## Entregáveis

- [ ]  Código-fonte - link repositório github

## Roteiro de Desenvolvimento

- [ ]  Etapa 1 – Escolha do tema.
- [ ]  Etapa 2 – Levantamento de requisitos.
- [ ]  Etapa 3 – Modelagem da solução.
- [ ]  Etapa 4 – Implementação OO (por exempo)

- [ ]  Etapa 5 – Implementação Funcional (por exemplo)
- [ ]  Etapa 6 – Testes.
- [ ]  Etapa 7 – Comparação dos paradigmas.
- [ ]  Etapa 8 – Apresentação final.

---

# Questões para o Relatório

- [ ]  Qual paradigma foi mais fácil de utilizar?
- [ ]  Qual paradigma gerou código mais legível?
- [ ]  Quais vantagens e limitações foram observadas?
- [ ]  Em quais situações cada paradigma seria mais indicado?
